# 와이어프레임 명세서: 화면 구조 및 상호작용

## 1. 문서 목적
- 본 문서는 MudBlazor 기반 UI의 화면 배치, 이동 경로, 컴포넌트 상태를 텍스트 와이어프레임으로 정의한다.
- 실제 픽셀 시안 대신 레이아웃·행동 중심 명세를 제공하여 구현과 검증의 공통 기준으로 사용한다.

## 2. 전역 레이아웃 구조
- 상단 AppBar: 메뉴 토글, 브랜드 타이틀, 다크모드 토글, 확장 아이콘 배치
- 좌측 Drawer: Home / Devices / DashBoard / VirtualFarm 네비게이션 링크 배치
- 본문 MainContent: 페이지별 카드/표/다이얼로그/차트 렌더링 영역
