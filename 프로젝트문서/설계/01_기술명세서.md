## 5. 비기능 요구사항
- NFR-001: UI 로딩 시 주요 대시보드 구성요소는 사용자가 즉시 인지 가능해야 한다.
- NFR-002: 장비 탐색 루프는 오류 장비를 별도 설명으로 노출해 운영자 판단을 돕는다.
- NFR-003: MQTT 연결 실패 시 치명 종료 대신 예외를 기록하고 화면 기능을 유지한다.
- NFR-004: Unity WebGL 리소스는 브라우저 캐시 정책과 압축 헤더를 함께 고려한다.
- NFR-005: 다이얼로그 기반 제어는 취소/확정 액션을 모두 제공한다.
- NFR-006: 차트 데이터는 실시간성보다 운영 가시성을 우선한다.
- NFR-007: 네트워크 설정 값(IP/포트)은 환경별로 분리 가능한 구조가 필요하다.
- NFR-008: CORS 설정은 운영 배포 시 제한 정책으로 대체 가능해야 한다.
- NFR-009: 로그 메시지는 장비 연결 상태와 오류 원인을 추적 가능하게 남겨야 한다.
- NFR-010: 문서 기준선은 구현 변경 시 함께 버전업되어야 한다.

## 6. 운영 시나리오별 기술 제약
- SCN-001: 사용자가 Devices 페이지에 진입하면 네트워크 스캔이 비동기로 시작된다. (추적코드 T11)
- SCN-002: 사용자가 장비 연결 아이콘을 클릭하면 로컬 리스트 상태가 Available→Connected로 이동한다. (추적코드 T12)
- SCN-003: 사용자가 Dashboard의 센서 카드 아이콘을 클릭하면 상세 제어 다이얼로그가 열린다. (추적코드 T13)
- SCN-004: 사용자가 VirtualFarm 페이지에서 전체화면을 요청하면 JS 함수가 Unity 인스턴스로 위임한다. (추적코드 T14)
